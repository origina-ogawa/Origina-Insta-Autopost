# Instagram カルーセル自動投稿システム 構想メモ

作成日: 2026-07-05

## 目的

- 自社発信用・他社代行発信用の2種類のテンプレートで、カルーセル投稿(添付サンプルのようなデザイン)を自動生成・自動投稿する
- 頻度: 平日のみ1日1回
- コスト: 無料〜数百円/月に収める

## 基本方針(最重要)

**画像そのものをAIに描かせず、「HTML/CSSテンプレート + AIが文章だけ生成 + ヘッドレスブラウザでスクリーンショット」方式にする。**

理由:
1. 画像生成AI(DALL-E等)は日本語の文字が高確率で崩れる。サンプルのような文字主体のデザインには不向き
2. 画像生成APIは1枚数円〜数十円かかる。カルーセル8枚×毎営業日だと月数千円になる
3. HTMLテンプレなら文字は絶対に崩れず、デザインも毎回一貫し、コストはゼロ
4. ブランドカラーやレイアウトをテンプレで固定でき、「自社用」「代行用」の切り替えはテンプレファイルの差し替えだけで済む

## 全体アーキテクチャ

```
[テーマ一覧 (topics.yml / Googleスプレッドシート)]
        │ ①今日のテーマを1つ取り出す
        ▼
[LLM API (Gemini Flash 無料枠 or Claude Haiku)]
        │ ②スライド構成の文章をJSONで生成
        │   (表紙タイトル、各スライドの見出し・本文・チェックリスト、キャプション、ハッシュタグ)
        ▼
[HTMLテンプレート (自社用 / 代行用)]
        │ ③JSONを流し込み → Playwright で 1080x1350 スクリーンショット
        ▼
[画像ホスティング (公開URL必須)]
        │ ④GitHub Pages or Cloudflare R2 無料枠にアップロード
        ▼
[Instagram API (公式・無料)]
        │ ⑤子コンテナ作成 → 親カルーセルコンテナ → publish
        ▼
[Instagram に投稿完了] → ⑥Chatworkに結果通知(任意)

すべてを GitHub Actions の cron (平日朝) で実行
```

## 各要素の選定理由とコスト

| 要素 | 採用技術 | コスト |
|---|---|---|
| 実行環境・スケジューラ | GitHub Actions (cron: 平日のみ) | 無料 (publicリポ、privateでも月2,000分無料) |
| 文章生成 | Gemini Flash (無料枠 1日1回なら余裕) or Claude Haiku | 0円 〜 月数十円 |
| 画像生成 | HTMLテンプレ + Playwright スクリーンショット | 無料 |
| 画像の公開URL | GitHub Pages or Cloudflare R2 (10GB無料) | 無料 |
| 投稿 | Instagram公式API (Content Publishing) | 無料 |
| 通知・承認 | Chatwork API | 無料 |

## Instagram API の前提条件・制約(規約まわり)

- **必ず公式APIを使う。** 非公式ライブラリ(instagrapi等)やブラウザ自動操作(Selenium等)での投稿は規約違反でアカウント凍結(BAN)のリスクが高い
- Instagramアカウントを**プロアカウント(ビジネス or クリエイター)**に切り替える必要がある
- Meta for Developers でアプリを作成し、アクセストークンを取得する
- API経由の投稿は **24時間あたり約100件が上限(カルーセルは1件扱い)** → 1日1件なら全く問題なし
- カルーセルは**最大10枚**。全画像は1枚目のアスペクト比にクロップされる(1080x1350 の 4:5 で統一推奨)
- 画像はAPIに直接アップロードできず、**公開URLで渡す**必要がある(ホスティングが必要な理由)

### 他社代行アカウントへの投稿について

- 代行先のInstagramアカウントもプロアカウント化が必要
- 代行先に自分のMetaアプリをOAuth認可してもらい、そのアカウント用のアクセストークンを取得する
- 正式にやるには Meta の**アプリ審査(Advanced Access)**が必要
- ただし、少数の固定クライアントなら、相手を自アプリの「テスター」ロールに追加すれば審査なしで運用開始できる(まず自社アカウントで実績を作ってから審査に出すのが現実的)

### スパム判定を避けるための運用ルール

- 毎回文面・テーマが変わるようにする(テーマリストで管理)
- 完全放置ではなく、投稿前にChatworkで下書きを確認する「承認フロー」を挟むのが安全
- ハッシュタグの使い回しすぎに注意(毎回少し変える)

## 実装フェーズ

1. **Phase 1: 手動投稿の疎通確認**
   - Instagramプロアカウント化、Metaアプリ作成、トークン取得
   - 手元の画像1枚をAPIで投稿できることを確認(ここが一番つまずきやすい)
2. **Phase 2: テンプレート作成**
   - 添付サンプルのデザインをHTML/CSSで再現(自社用)
   - Playwright でスクリーンショット → 1080x1350 PNG が出ることを確認
3. **Phase 3: 文章の自動生成**
   - テーマリスト(topics.yml)を作る
   - LLMにJSON形式でスライド文章を生成させるプロンプトを作る
4. **Phase 4: 自動化**
   - GitHub Actions で cron 実行(例: `0 22 * * 0-4` = JST平日朝7時)
   - 画像アップロード → カルーセル投稿までを一気通貫
5. **Phase 5: 運用強化**
   - Chatwork承認フロー(生成→通知→OKリアクションで投稿)
   - 代行用テンプレ追加、複数アカウント対応

## スライドごとにデザインを変える方法(コンポーネント方式)

2026-07-05 検証済み。固定レイアウト1枚ではなく、**「部品(コンポーネント)の組み合わせ」をAIに選ばせる**方式にする。

- テンプレートは「表紙」「本文」「まとめ」の3レイアウト + 本文用の部品集(チェックリスト / 比較ブロック≠ / 吹き出し / 引用 / グラフ風 など)で構成する
- アイコンは **Tabler Icons(無料・5,800種以上)** をWebフォントとして読み込み、AIがJSON内で `"icon": "ti-lock"` のようにアイコン名を指定する → 文章の内容に合ったアイコンが自動で入る
- AIの出力JSON例:

```json
{
  "slide_type": "body",
  "number": "02",
  "title": "相場より極端に安いを売りにしている会社",
  "blocks": [
    { "type": "warning_lead", "icon": "ti-alert-circle", "text": "..." },
    { "type": "compare", "left": {"icon": "ti-template", "label": "テンプレ量産"}, "right": {"icon": "ti-target-arrow", "label": "戦略設計"} },
    { "type": "checklist", "items": [ {"icon": "ti-lock", "text": "...", "em": "サイトの所有権がない"} ] }
  ]
}
```

- 検証結果: `templates/demo-slide.html` → `templates/demo-slide.png`(1080x1080)。サンプルデザインをほぼ再現できることを確認済み
- さらに自由度が欲しい場合は「CSSデザインシステムは固定、HTML構造はAIが丸ごと生成」する方式もある(レイアウト崩れのリスクがあるので承認フロー必須)

## 画像生成AIを使う場合のコスト(参考・非推奨)

- Gemini 2.5 Flash Image (nano banana): 無料枠あり(約500リクエスト/日)。有料でも1枚 $0.039〜0.067
- 8枚 × 月22営業日 = 176枚 → 約 $7〜12/月(1,100〜1,800円)
- ただし日本語文字の崩れ・スライド間のデザイン一貫性の問題が残るため、文字主体のデザインには非推奨

## 技術スタックの選択肢

- **Node.js + Playwright(推奨)**: スクリーンショットが最も簡単・安定。GitHub Actionsとの相性も良い
- Ruby でも可(ferrum gem でChrome操作)。Rails経験を活かすならこちらだが、今回はWebアプリ不要なのでスクリプトだけで完結する

## 注意事項

- アクセストークン等の機密情報は GitHub Secrets に保存し、`.env` はリポジトリに含めない(`.gitignore` 必須)
- 長期トークン(60日有効)は定期的に更新が必要 → 更新処理も自動化対象
